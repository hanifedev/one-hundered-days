## üìñ Task Explanations

Today, you'll learn about Terraform variables and outputs, which make your configurations more flexible and reusable. Variables allow you to parameterize your infrastructure, while outputs provide important information after deployment.

Variables in Terraform:
- Make configurations reusable (same code, different values)
- Improve maintainability (change values in one place)
- Allow customization without modifying code
- Support different environments (dev, staging, prod)

You'll define variables with types (string, number, bool, list, map) and optionally provide default values. You can pass variable values via:
- Command line (`-var` flag)
- Variable files (`.tfvars`)
- Environment variables
- Default values in variable definitions

Outputs are useful for:
- Displaying important information (IP addresses, resource IDs)
- Passing values to other Terraform configurations
- Providing information to users or other systems

You'll refactor a previous configuration to use variables, making it more flexible and maintainable.

## üìö Dictionary for Developers

*   **Terraform Variable:** A parameter in Terraform that allows you to customize configuration without modifying code.
*   **Variable Definition:** The declaration of a variable in Terraform, including its type, description, and optional default value.
*   **`.tfvars` File:** A file containing variable values, used to pass values to Terraform configurations.
*   **Terraform Output:** A value exported from a Terraform configuration after it's applied, useful for displaying important information.
*   **Parameterized Configuration:** A Terraform configuration that uses variables to make it flexible and reusable.
