## üìñ Task Explanations

Today, you'll learn about Continuous Deployment (CD), which automates the process of deploying code to production or staging environments after it passes CI checks.

CD extends CI by:
- Automatically deploying code that passes tests
- Reducing manual deployment errors
- Enabling faster releases
- Ensuring consistent deployments

You'll learn about deployment strategies:
- **Direct Deployment:** Deploy directly to production
- **Blue-Green:** Maintain two identical environments, switch between them
- **Canary:** Deploy to a small subset of users first
- **Rolling Updates:** Gradually update instances

You'll create a deployment workflow that uses GitHub Secrets to store sensitive credentials securely. This is crucial for real-world deployments where you need API keys, passwords, or tokens.

You'll practice by deploying a simple application, giving you hands-on experience with the deployment process.

## üìö Dictionary for Developers

*   **Continuous Deployment (CD):** The practice of automatically deploying code to production after it passes CI checks.
*   **Deployment Strategy:** A method for deploying applications (e.g., blue-green, canary, rolling updates) to minimize risk and downtime.
*   **GitHub Secrets:** Encrypted variables stored in GitHub that can be used in workflows without exposing sensitive information.
*   **Staging Environment:** A production-like environment used for testing before deploying to production.
*   **Automated Deployment:** The process of deploying applications automatically through CI/CD pipelines, without manual intervention.
