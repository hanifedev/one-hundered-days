## üìñ Task Explanations

Today, you'll learn about Google Cloud Platform (GCP), the third major cloud provider. GCP is known for its strengths in data analytics, machine learning, and Kubernetes (Google created Kubernetes).

GCP core concepts:
- **Projects:** Top-level containers for organizing resources
- **Compute Engine:** Virtual machines (equivalent to EC2)
- **Cloud Storage:** Object storage (equivalent to S3)
- **Cloud Functions:** Serverless compute (equivalent to Lambda)
- **GKE:** Google Kubernetes Engine (managed Kubernetes)

You'll explore the GCP Console, which has a different interface than AWS and Azure. Understanding multiple interfaces makes you adaptable.

You'll create basic resources:
- Create a Project
- Create a Compute Engine instance or Cloud Storage bucket
- Understand GCP's resource model

You'll compare all three major cloud providers:
- **AWS:** Largest, most services, enterprise-focused
- **Azure:** Strong Microsoft integration, hybrid cloud
- **GCP:** Strong in data/ML, Kubernetes expertise, developer-friendly

Understanding all three makes you truly cloud-agnostic and valuable.

## üìö Dictionary for Developers

*   **GCP (Google Cloud Platform):** Google's cloud computing platform, known for data analytics and Kubernetes expertise.
*   **GCP Project:** A top-level container in GCP that organizes resources and provides isolation and billing.
*   **Compute Engine:** GCP's virtual machine service, equivalent to AWS EC2 and Azure Virtual Machines.
*   **Cloud Storage:** GCP's object storage service, equivalent to AWS S3 and Azure Blob Storage.
*   **Cloud Functions:** GCP's serverless compute service, equivalent to AWS Lambda and Azure Functions.
*   **GKE (Google Kubernetes Engine):** GCP's managed Kubernetes service, known for being the original Kubernetes provider.
