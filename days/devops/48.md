## üìñ Task Explanations

Today, you'll learn about Ansible roles, which are a way to organize and reuse playbook code. Roles make Ansible code more maintainable and shareable.

Roles provide:
- **Organization:** Group related tasks together
- **Reusability:** Use the same role in multiple playbooks
- **Sharing:** Share roles via Ansible Galaxy
- **Modularity:** Break complex playbooks into manageable pieces

Role structure:
- **tasks/main.yml:** Main tasks for the role
- **handlers/main.yml:** Handlers used by the role
- **templates/:** Jinja2 templates for configuration files
- **vars/main.yml:** Role variables
- **defaults/main.yml:** Default variable values
- **files/:** Static files to copy

You'll create a role, perhaps for:
- Setting up a web server (Nginx, Apache)
- Configuring a database
- Installing and configuring an application

You'll use the role in a playbook, seeing how roles simplify and organize your automation code.

You'll also learn best practices for writing maintainable Ansible code.

## üìö Dictionary for Developers

*   **Ansible Role:** A way to organize and package Ansible code into reusable units with a standard directory structure.
*   **Role Structure:** The standard directory layout for Ansible roles (tasks, handlers, templates, vars, defaults, files).
*   **Ansible Galaxy:** A repository for sharing Ansible roles and collections.
*   **Jinja2 Template:** A templating engine used by Ansible to generate configuration files with variables.
*   **Role Reusability:** The ability to use the same role in multiple playbooks or projects.
*   **Configuration Management Best Practices:** Guidelines for writing maintainable, efficient, and secure Ansible code.
