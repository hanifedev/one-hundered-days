## üìñ Task Explanations

Today, you'll learn about Microsoft Azure, which is the second-largest cloud provider after AWS. Understanding multiple cloud platforms makes you more versatile and valuable as a DevOps professional.

Azure core concepts:
- **Resource Groups:** Containers for organizing related resources
- **Virtual Machines:** Azure's compute service (equivalent to EC2)
- **Blob Storage:** Object storage (equivalent to S3)
- **Azure Functions:** Serverless compute (equivalent to Lambda)
- **App Service:** Platform-as-a-Service for web apps

You'll explore the Azure Portal, which is the web interface for managing Azure resources. Understanding the portal layout will help you navigate and find services.

You'll create basic resources to get hands-on experience:
- Create a Resource Group
- Create a Virtual Machine or Storage Account
- Understand Azure's resource model

You'll compare Azure with AWS:
- Similar services with different names
- Different approaches to some concepts
- Understanding both makes you cloud-agnostic

This knowledge expands your cloud expertise beyond AWS.

## üìö Dictionary for Developers

*   **Azure (Microsoft Azure):** Microsoft's cloud computing platform, providing services similar to AWS.
*   **Resource Group:** An Azure container that holds related resources for an application, used for organization and management.
*   **Azure Virtual Machine:** Azure's compute service for running virtual machines, equivalent to AWS EC2.
*   **Azure Blob Storage:** Azure's object storage service, equivalent to AWS S3.
*   **Azure Functions:** Azure's serverless compute service, equivalent to AWS Lambda.
*   **Azure Portal:** The web-based interface for managing Azure resources and services.
