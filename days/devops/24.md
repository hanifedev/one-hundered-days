## üìñ Task Explanations

Today, you'll learn about IAM, which is fundamental to AWS security. IAM controls who can access what in your AWS account. Proper IAM configuration is crucial for security and is a core DevOps responsibility.

IAM concepts:
- **Users:** Individual people or applications that need access
- **Groups:** Collections of users with similar permissions
- **Roles:** Temporary credentials for services or users
- **Policies:** JSON documents that define permissions

You'll create an IAM user instead of using the root account. This is a security best practice‚Äîthe root account should rarely be used. IAM users can have specific permissions, making it easier to manage access and audit who did what.

You'll write IAM policies, which are JSON documents that define what actions are allowed or denied. Policies follow the principle of least privilege‚Äîgranting only the minimum permissions needed.

Understanding IAM is essential for managing access in AWS and is a critical skill for DevOps professionals.

## üìö Dictionary for Developers

*   **IAM (Identity and Access Management):** AWS service for managing user access and permissions to AWS resources.
*   **IAM User:** An identity in AWS that represents a person or application and can have specific permissions.
*   **IAM Policy:** A JSON document that defines permissions (what actions are allowed or denied on which resources).
*   **IAM Group:** A collection of IAM users, useful for managing permissions for multiple users at once.
*   **IAM Role:** A set of permissions that can be assumed by users, applications, or AWS services temporarily.
*   **Principle of Least Privilege:** A security best practice of granting only the minimum permissions necessary to perform a task.
*   **Root Account:** The original AWS account with full access. Should be protected and rarely used.
