## üìñ Task Explanations

Today, you'll learn Docker Compose, which simplifies running multi-container applications. While `docker run` works for single containers, real applications often need multiple containers (web server, database, cache, etc.) working together.

Docker Compose uses a YAML file (`docker-compose.yml`) to define:
- **Services:** The containers that make up your application
- **Networks:** How services communicate
- **Volumes:** Persistent data storage
- **Environment Variables:** Configuration for services

You'll create a compose file for a simple multi-container setup (e.g., a web app and database). This will help you understand how to orchestrate containers and manage complex applications.

Docker Compose commands are straightforward:
- `docker-compose up`: Start all services
- `docker-compose down`: Stop and remove all services
- `docker-compose ps`: List running services
- `docker-compose logs`: View logs from services

## üìö Dictionary for Developers

*   **Docker Compose:** A tool for defining and running multi-container Docker applications using a YAML file.
*   **`docker-compose.yml`:** A YAML file that defines services, networks, and volumes for a Docker Compose application.
*   **Service:** A container definition in a Docker Compose file, representing one component of the application.
*   **Network:** A Docker network that allows containers to communicate with each other.
*   **Volume:** A mechanism for persisting data generated by containers, separate from the container's lifecycle.
*   **`docker-compose up`:** Command to create and start all services defined in the compose file.
*   **`docker-compose down`:** Command to stop and remove all services, networks, and volumes.
