## üìñ Task Explanations

Today, you'll learn about AWS VPC, which provides network isolation and control in AWS. Understanding VPC is crucial for building secure, properly architected cloud applications.

VPC concepts:
- **VPC:** Your own isolated network in AWS
- **Subnet:** A range of IP addresses in your VPC (public or private)
- **Route Table:** Defines how traffic is routed
- **Internet Gateway:** Allows resources in public subnets to access the internet
- **NAT Gateway:** Allows resources in private subnets to access the internet (outbound only)

You'll create a VPC with:
- **Public subnets:** For resources that need internet access (load balancers, bastion hosts)
- **Private subnets:** For resources that shouldn't be directly accessible from the internet (databases, application servers)

This architecture provides security by isolating sensitive resources while still allowing necessary connectivity. Understanding VPC is essential for production AWS deployments.

## üìö Dictionary for Developers

*   **VPC (Virtual Private Cloud):** An isolated network environment in AWS that you can configure and control.
*   **Subnet:** A range of IP addresses in your VPC. Can be public (internet-accessible) or private (isolated).
*   **Route Table:** A set of rules that determine where network traffic is directed.
*   **Internet Gateway:** A VPC component that allows communication between resources in your VPC and the internet.
*   **NAT Gateway:** A network address translation service that allows resources in private subnets to access the internet (outbound only).
*   **Network Isolation:** Separating resources into different network segments for security and organization.
