## ğŸ“– Task Explanations

Today, you'll dive into Kubernetes basics by working with pods, the fundamental unit in Kubernetes. Understanding pods is crucialâ€”everything in Kubernetes builds on pods.

You'll learn essential `kubectl` commands:
- **`kubectl get`:** List resources (pods, nodes, services)
- **`kubectl describe`:** Get detailed information about a resource
- **`kubectl create`:** Create resources from command line
- **`kubectl apply`:** Apply configuration from YAML files
- **`kubectl delete`:** Delete resources

You'll create your first pod, which is the simplest Kubernetes resource. A pod can contain one or more containers that work together. You'll create pods using both command-line and YAML files.

Understanding pod lifecycle is important:
- **Pending:** Pod is being created
- **Running:** Pod is running
- **Succeeded/Failed:** Pod has completed or failed

You'll also learn to view logs from pods, which is essential for debugging and monitoring.
