## üìñ Task Explanations

Today, you'll be introduced to Infrastructure as Code (IaC), which is a fundamental DevOps practice. IaC allows you to define and manage infrastructure using code, rather than manually configuring resources.

Benefits of IaC:
- **Version Control:** Infrastructure changes are tracked in Git
- **Reproducibility:** Same code creates the same infrastructure
- **Consistency:** Eliminates manual configuration errors
- **Documentation:** Code serves as documentation

You'll learn about Terraform, which is one of the most popular IaC tools. Terraform uses a declarative language (HCL - HashiCorp Configuration Language) to define infrastructure.

Terraform concepts:
- **Providers:** Plugins that interact with cloud providers (AWS, Azure, GCP)
- **Resources:** Infrastructure components (EC2 instances, S3 buckets, etc.)
- **State:** Terraform's record of what it has created
- **Configuration Files:** `.tf` files that define your infrastructure

You'll write your first Terraform configuration, which will define a simple resource. This hands-on experience will help you understand the basic structure.

## üìö Dictionary for Developers

*   **Infrastructure as Code (IaC):** The practice of managing and provisioning infrastructure through code instead of manual processes.
*   **Terraform:** An open-source IaC tool by HashiCorp that uses declarative configuration files to manage infrastructure.
*   **Provider:** A plugin in Terraform that interacts with a specific cloud provider or service (AWS, Azure, GCP, etc.).
*   **Resource:** A component of infrastructure defined in Terraform (e.g., an EC2 instance, S3 bucket).
*   **Terraform State:** A file that tracks the resources Terraform has created and their current state.
*   **HCL (HashiCorp Configuration Language):** The configuration language used by Terraform to define infrastructure.
