## ðŸ“– Task Explanations

Today, you'll learn about Auto Scaling, which automatically adjusts the number of instances based on demand. This is essential for handling traffic spikes, optimizing costs, and maintaining performance.

Auto Scaling benefits:
- **Cost optimization:** Scale down during low traffic, scale up during high traffic
- **Handling spikes:** Automatically add capacity when traffic increases
- **High availability:** Replace failed instances automatically
- **Performance:** Maintain performance under varying loads

Auto Scaling concepts:
- **Launch Template:** Defines the configuration for new instances (AMI, instance type, security groups)
- **Auto Scaling Group:** A group of instances that Auto Scaling manages
- **Scaling Policy:** Rules that determine when to scale (e.g., CPU > 70%)
- **Scaling Triggers:** Metrics that trigger scaling actions

You'll create an Auto Scaling Group that:
- Maintains a minimum number of instances
- Scales up when demand increases
- Scales down when demand decreases
- Replaces unhealthy instances

You'll configure scaling policies based on CloudWatch metrics, creating a self-managing, scalable infrastructure.

## ðŸ“š Dictionary for Developers

*   **Auto Scaling:** The automatic adjustment of compute capacity based on demand, adding or removing instances as needed.
*   **Auto Scaling Group:** A group of EC2 instances that Auto Scaling manages, automatically adjusting the number of instances.
*   **Launch Template:** A configuration template that defines how new instances are created (AMI, instance type, security groups, etc.).
*   **Scaling Policy:** A rule that determines when and how to scale (e.g., add instances when CPU > 70%).
*   **Scaling Trigger:** A metric or condition that causes Auto Scaling to take action (scale up or down).
*   **Dynamic Scaling:** Automatically adjusting capacity based on real-time demand, as opposed to fixed capacity.
