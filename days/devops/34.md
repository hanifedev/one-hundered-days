## üìñ Task Explanations

Today, you'll learn about ConfigMaps and Secrets, which are essential for managing configuration and sensitive data in Kubernetes.

**ConfigMaps** store non-sensitive configuration data:
- Application settings
- Configuration files
- Environment-specific values

They allow you to separate configuration from application code, making applications more flexible and easier to manage across different environments.

**Secrets** store sensitive data:
- Passwords
- API keys
- TLS certificates
- Database credentials

Secrets are similar to ConfigMaps but are designed for sensitive data. While they're not encrypted by default (base64 encoded), they're treated specially by Kubernetes.

You'll learn to:
- Create ConfigMaps and Secrets
- Mount them as environment variables in pods
- Mount them as files in pods
- Update them and see changes reflected in pods

This knowledge is crucial for managing configuration in production Kubernetes deployments.

## üìö Dictionary for Developers

*   **ConfigMap:** A Kubernetes resource that stores non-sensitive configuration data as key-value pairs.
*   **Secret:** A Kubernetes resource that stores sensitive data (passwords, API keys, certificates) in an encoded format.
*   **Configuration Management:** The practice of managing application configuration separately from code, allowing flexibility across environments.
*   **Environment Variables:** Variables that can be injected into pods from ConfigMaps or Secrets.
*   **Volume Mount:** Attaching ConfigMaps or Secrets as files in pods, allowing applications to read configuration files.
