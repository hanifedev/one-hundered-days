## ğŸ“– Task Explanations

Today, you'll learn how to actually use Provider data in your widgets. The `Consumer` widget is one way to access Provider data. It automatically rebuilds its child widget whenever the Provider's state changes.

However, `Consumer` rebuilds whenever ANY part of the state changes, which can be inefficient if your state object has many properties and you only care about one of them. That's where `Selector` comes inâ€”it allows you to specify exactly which part of the state you want to listen to, and it only rebuilds when that specific part changes.

Understanding when widgets rebuild is crucial for performance. You want to minimize rebuilds to keep your app fast and responsive. `Selector` is a powerful tool for this optimization.

## ğŸ“š Dictionary for Developers

*   **`Consumer`:** A widget that listens to a Provider and rebuilds its child whenever the Provider's state changes.
*   **`Selector`:** A widget that listens to a Provider but only rebuilds when a specific part of the state changes, improving performance.
*   **Widget Rebuild:** The process of Flutter recreating a widget's UI when its state or dependencies change.
*   **Performance Optimization:** Techniques used to improve app speed and efficiency, such as minimizing unnecessary widget rebuilds.
*   **`Provider.of()`:** An alternative way to access Provider data, but it rebuilds the entire widget, which may be less efficient than `Consumer` or `Selector`.
