## üìñ Task Explanations

Today, you'll learn about memory management and optimization. Memory issues can cause apps to slow down, crash, or drain battery. Understanding how to identify and fix memory problems is crucial for building stable apps.

The Memory tab in DevTools shows:
- Memory usage over time
- Object allocations
- Garbage collection events
- Memory leaks (objects that should be freed but aren't)

Common memory issues include:
- **Memory Leaks:** Objects that aren't properly disposed (e.g., listeners, streams, controllers)
- **Excessive Allocations:** Creating too many objects unnecessarily
- **Large Objects:** Holding references to large objects longer than needed

You'll learn optimization techniques:
- Use `const` widgets to prevent unnecessary rebuilds
- Use `RepaintBoundary` to isolate expensive repaints
- Properly dispose of controllers, streams, and listeners
- Avoid creating objects in `build()` methods

You'll practice by analyzing a previous project and identifying memory issues, then fixing them.

## üìö Dictionary for Developers

*   **Memory Leak:** A situation where objects are not properly freed from memory, causing memory usage to grow over time.
*   **Garbage Collection:** The automatic process of freeing memory that's no longer in use.
*   **Memory Profiling:** The process of analyzing memory usage to identify leaks and optimization opportunities.
*   **`RepaintBoundary`:** A widget that isolates repaints, preventing expensive repaints from affecting the entire widget tree.
*   **Memory Optimization:** Techniques for reducing memory usage and preventing memory leaks.
