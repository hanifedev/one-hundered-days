## üìñ Task Explanations

Today, you'll learn about theming, which is essential for creating consistent, professional-looking applications. A theme defines the visual style of your app: colors, text styles, button styles, and more.

`ThemeData` is Flutter's theming system. It provides a comprehensive set of properties that control the appearance of Material Design components. Instead of hardcoding colors and styles throughout your app, you define them once in a theme and use them consistently.

You'll set a global theme using `MaterialApp`'s `theme` parameter. This theme will be used by all Material widgets in your app automatically. You can access theme properties in your widgets using `Theme.of(context)`, which allows you to use theme colors and styles in custom widgets.

Customizing your theme involves setting properties like:
- `colorScheme`: Primary, secondary, error colors, etc.
- `textTheme`: Text styles for different text types
- `buttonTheme`, `cardTheme`, etc.: Styles for specific components

## üìö Dictionary for Developers

*   **Theming:** The practice of defining a consistent visual style (colors, fonts, styles) for an application.
*   **`ThemeData`:** Flutter's class that defines the visual properties of Material Design components in your app.
*   **Global Theme:** A theme applied to the entire app, ensuring consistent styling across all screens and widgets.
*   **`Theme.of(context)`:** A method that retrieves the current theme from the widget tree, allowing widgets to use theme properties.
*   **Color Scheme:** A set of colors (primary, secondary, error, etc.) that define the color palette for your app.
