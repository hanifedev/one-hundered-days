## üìñ Task Explanations

Today, you'll be introduced to CI/CD, which is essential for professional development workflows. CI/CD automates testing, building, and deploying your app, ensuring consistency and catching issues early.

**Continuous Integration (CI):** Automatically builds and tests your code whenever you push changes. This catches bugs early and ensures code quality.

**Continuous Deployment (CD):** Automatically deploys your app when changes pass all tests. This speeds up releases and reduces manual errors.

GitHub Actions is a popular CI/CD platform that's free for public repositories. You'll create a workflow file (YAML format) that defines:
- When to run (on push, pull request, etc.)
- What to do (install dependencies, run tests, build app)
- Where to deploy (if applicable)

You'll set up a basic workflow that:
- Checks out your code
- Sets up Flutter
- Installs dependencies
- Runs tests
- Builds your app

This ensures your app builds successfully and tests pass on every change.

## üìö Dictionary for Developers

*   **CI/CD (Continuous Integration/Continuous Deployment):** Automated processes that build, test, and deploy applications.
*   **GitHub Actions:** A CI/CD platform integrated with GitHub that automates workflows.
*   **Workflow:** An automated process defined in a YAML file that runs on specific events (push, PR, etc.).
*   **Continuous Integration:** Automatically building and testing code whenever changes are pushed.
*   **Automated Testing:** Running tests automatically as part of the CI/CD pipeline, catching bugs early.
