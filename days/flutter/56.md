## üìñ Task Explanations

Today, you'll learn about local data storage, which is essential for building apps that work offline and provide fast access to data. While APIs are great for fetching data from servers, storing data locally allows your app to work without internet and provides better performance.

Flutter offers several options for local storage:
- `shared_preferences`: Simple key-value storage for small data
- Hive/Isar: Fast, lightweight NoSQL databases, great for structured data
- SQLite: Powerful relational database, ideal for complex data relationships

For this learning path, we'll focus on Hive or SQLite as they're the most common choices for structured data storage. Hive is simpler and faster for most use cases, while SQLite offers more power for complex queries.

You'll set up your chosen database and initialize it in your app. This typically involves initializing the database when your app starts and ensuring it's ready to use.

## üìö Dictionary for Developers

*   **Local Storage:** Storing data on the device itself rather than on a remote server, allowing offline access and faster performance.
*   **NoSQL Database:** A database that stores data in a non-relational format (like key-value pairs or documents), often simpler and faster than SQL databases.
*   **SQLite:** A lightweight, file-based SQL database engine that's widely used in mobile development.
*   **Hive:** A fast, lightweight NoSQL database for Flutter that stores data in key-value format.
*   **Isar:** A modern, high-performance NoSQL database for Flutter, similar to Hive but with more features.
*   **Offline Capability:** The ability of an app to function without an internet connection by using locally stored data.
