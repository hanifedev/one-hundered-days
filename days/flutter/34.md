## üìñ Task Explanations

Today, you'll add advanced features to your To-Do app and improve its robustness. Editing functionality will require you to manage an "editing mode" state, where a todo can be modified. This is a great exercise in managing more complex state scenarios.

Filtering is another important feature that will help you understand how to derive UI from state. You'll maintain a filter state (e.g., "all", "completed", "active") and use it to filter the list of todos before displaying them.

If you choose to add persistence, you'll get your first taste of working with external packages in Flutter. The `shared_preferences` package allows you to store simple data locally on the device, which means your todos will survive app restarts.

Finally, as your app grows, code organization becomes crucial. Extracting widgets into separate files and organizing your code logically will make it more maintainable and easier to understand.

## üìö Dictionary for Developers

*   **State Derivation:** The process of computing UI or data from existing state (e.g., filtering a list based on a filter state).
*   **Persistence:** The ability to save data so it remains available after the application is closed and reopened.
*   **`shared_preferences`:** A Flutter package for storing simple key-value pairs locally on the device.
*   **Code Refactoring:** The process of restructuring existing code without changing its functionality to improve readability and maintainability.
