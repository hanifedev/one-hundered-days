## üìñ Task Explanations

Today, you'll learn about advanced navigation patterns: nested navigation and route guards. These are common in real-world applications.

Nested navigation is when you have navigation within navigation. For example, a bottom navigation bar with multiple tabs, where each tab has its own navigation stack. This is common in apps like Instagram or Twitter, where you can navigate within each tab independently.

`go_router` provides `ShellRoute` for this purpose. It allows you to create a shell (like a scaffold with bottom navigation) that contains multiple child routes, each with its own navigation stack.

Route guards protect routes by checking conditions before allowing navigation. For example:
- Authentication guard: Check if user is logged in before accessing protected routes
- Role guard: Check if user has the right permissions
- Data guard: Ensure required data is loaded

You'll implement redirects that automatically send users to the right place based on their state (e.g., redirect to login if not authenticated, redirect to home if already logged in).

## üìö Dictionary for Developers

*   **Nested Navigation:** Navigation within navigation, such as tabs with their own navigation stacks.
*   **`ShellRoute`:** A route type in `go_router` that creates a shell (container) for nested navigation.
*   **Route Guard:** A mechanism that checks conditions before allowing navigation to a route (e.g., authentication check).
*   **Redirect:** Automatic navigation to a different route based on conditions (e.g., redirect to login if not authenticated).
*   **Navigation Stack:** A stack of screens that users can navigate through, with back navigation support.
