## üìñ Task Explanations

Today, you'll be introduced to animations in Flutter, which are essential for creating smooth, polished user experiences. Animations provide visual feedback, guide user attention, and make apps feel more responsive and professional.

Flutter offers two main types of animations:
- **Implicit Animations:** Automatic animations that happen when properties change (e.g., `AnimatedContainer`, `AnimatedOpacity`)
- **Explicit Animations:** Manual animations with full control over the animation (covered later)

You'll start with implicit animations, which are simpler and perfect for most use cases. `AnimatedContainer` is a great starting point‚Äîit's like a regular `Container`, but it automatically animates when you change its properties (color, size, padding, etc.).

You'll control animation speed using the `duration` parameter, which specifies how long the animation should take. You can also use `curve` to control the animation's timing (e.g., ease-in, ease-out).

## üìö Dictionary for Developers

*   **Animation:** A visual effect that creates the illusion of motion or change over time, making UI transitions smooth and polished.
*   **Implicit Animation:** An automatic animation that occurs when a widget's properties change, requiring minimal code.
*   **`AnimatedContainer`:** A widget that automatically animates changes to its properties (size, color, padding, etc.).
*   **Animation Duration:** The length of time an animation takes to complete, typically specified in milliseconds or seconds.
*   **Animation Curve:** A function that defines how an animation progresses over time (e.g., linear, ease-in, ease-out).
