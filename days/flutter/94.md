## ğŸ“– Task Explanations

Today, you'll learn about event channels, which are used when you need continuous data from native code (as opposed to one-time method calls). Event channels create a stream of data that flows from native code to Dart.

Use event channels when:
- You need continuous updates (e.g., battery level, sensor readings, location updates)
- Native code needs to push data to Dart over time
- You're listening to platform events (e.g., network changes, location updates)

The flow works like this:
1. Dart sets up an `EventChannel` and listens to the stream
2. Native code starts sending events
3. Events flow through the channel to Dart
4. Dart handles events in the stream

This is different from method channels, which are for one-time calls and responses. Event channels are for ongoing communication.

You'll implement a simple example, such as listening to battery level changes or sensor data, to understand how event channels work in practice.

## ğŸ“š Dictionary for Developers

*   **Event Channel:** A platform channel that provides a continuous stream of data from native code to Dart.
*   **Stream:** A sequence of asynchronous events that can be listened to, used by event channels to send data.
*   **Continuous Data:** Data that updates over time (e.g., sensor readings, location updates, battery level).
*   **Platform Events:** Events that originate from the platform (Android/iOS) and need to be communicated to Flutter.
*   **One-Way Communication:** Data flow from native code to Dart (as opposed to method channels which are two-way).
