## üìñ Task Explanations

Today, you'll learn about Hero animations, which are one of Flutter's most impressive animation features. Hero animations create smooth transitions where a widget appears to "fly" from one screen to another, creating a sense of continuity and polish.

Hero animations work by matching widgets with the same `tag` across different screens. When you navigate from one screen to another, Flutter automatically animates the transition between the two Hero widgets, creating a seamless visual connection.

Common use cases include:
- Images that expand from a list to a detail view
- Avatars that expand to show profile details
- Cards that expand to show full content

You'll wrap the widget you want to animate with a `Hero` widget and provide a unique tag. The same tag must be used on both screens. Flutter handles the rest automatically, creating a smooth transition.

## üìö Dictionary for Developers

*   **Hero Animation:** A type of animation where a widget appears to transition smoothly from one screen to another, creating visual continuity.
*   **`Hero` Widget:** A widget that enables Hero animations by matching widgets with the same tag across screens.
*   **Hero Tag:** A unique identifier used to match Hero widgets across different screens for animation.
*   **Shared Element Transition:** An animation where a single element appears to move from one screen to another, commonly used in mobile apps.
*   **Visual Continuity:** The sense that UI elements are connected and flow smoothly, created by animations like Hero.
