## üìñ Task Explanations

Today, you'll learn how to work with multiple providers in your app. Real-world applications typically have multiple pieces of state that need to be managed separately (e.g., user authentication, shopping cart, theme preferences). You'll learn how to set up and use multiple providers.

Flutter's Provider package offers different types of providers for different use cases:
- `Provider`: For simple, immutable data
- `ChangeNotifierProvider`: For mutable state that changes over time
- `FutureProvider`: For data that comes from async operations (like API calls)
- `StreamProvider`: For data that comes from streams

You'll also learn about `MultiProvider`, which allows you to provide multiple providers at once, making your code cleaner when you have many providers.

## üìö Dictionary for Developers

*   **`MultiProvider`:** A widget that allows you to provide multiple providers at once, useful when you have many providers at the root level.
*   **`FutureProvider`:** A Provider type that provides data from a Future, automatically handling loading and error states.
*   **`StreamProvider`:** A Provider type that provides data from a Stream, automatically updating when new data arrives.
*   **Provider Scope:** The part of the widget tree where a Provider is available. Providers are available to all descendant widgets.
*   **State Separation:** The practice of dividing application state into logical, separate providers for different features or domains.
