## üìñ Task Explanations

Today, you'll dive deeper into making GET requests and learn how to properly handle asynchronous operations. HTTP requests take time to complete (network latency), so they're asynchronous operations. You'll use `async`/`await` to wait for the response before continuing.

It's important to check the status code of the response. A status code of 200 means the request was successful, while codes like 404 (not found) or 500 (server error) indicate problems. Always check the status before trying to use the response data.

The response body is typically a JSON string, which you'll need to parse (convert from string to Dart objects) in the coming days. For now, you'll work with the raw string to understand the structure. You'll also learn how to integrate this data fetching into your UI, which requires understanding how to update the UI when asynchronous data arrives.

## üìö Dictionary for Developers

*   **Asynchronous Operation:** An operation that doesn't block the execution of other code. HTTP requests are asynchronous because they take time to complete.
*   **Status Code:** A three-digit number returned by the server indicating the result of an HTTP request (200 = success, 404 = not found, 500 = server error).
*   **Response Body:** The actual data content returned by the server, typically in JSON format for REST APIs.
*   **Endpoint:** A specific URL that represents a resource or service provided by an API.
