## üìñ Task Explanations

Today, you'll learn about JSON, which is the most common format for API responses. JSON is a text-based data format that's easy for both humans and machines to read. Understanding JSON is crucial because almost all modern APIs use it.

You'll start with manual JSON parsing using Dart's built-in `dart:convert` library. This involves using `jsonDecode()` to convert a JSON string into Dart objects (typically `Map<String, dynamic>` for JSON objects or `List<dynamic>` for JSON arrays).

While manual parsing works, it can be error-prone and verbose. However, understanding how it works is important before moving on to automated solutions. You'll learn how to access nested data, handle arrays, and work with different JSON data types.

## üìö Dictionary for Developers

*   **JSON (JavaScript Object Notation):** A lightweight data-interchange format that is easy for humans to read and write and easy for machines to parse and generate.
*   **JSON Parsing:** The process of converting a JSON string into native programming language objects (in this case, Dart objects).
*   **`jsonDecode()`:** A function from Dart's `dart:convert` library that converts a JSON string into Dart objects (Map, List, etc.).
*   **`Map<String, dynamic>`:** A Dart map type that can hold key-value pairs where keys are strings and values can be any type (dynamic).
*   **Nested JSON:** JSON structure where objects or arrays are contained within other objects or arrays, creating a hierarchical structure.
