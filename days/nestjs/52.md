## ğŸ“š Dictionary for Developers

*   **Clean Architecture:** An architectural approach that organizes code into layers with clear boundaries and dependency rules.
*   **Dependency Rule:** The principle that dependencies should point inward (toward the domain layer).
*   **Domain Layer:** The innermost layer containing business logic and entities.
*   **Application Layer:** The layer containing use cases and application-specific logic.
*   **Infrastructure Layer:** The outermost layer containing framework and database code.
