## ğŸ“– Task Explanations

WebSockets enable real-time, bidirectional communication between client and server. This is essential for features like chat, live notifications, collaborative editing, or real-time dashboards. Unlike HTTP requests, WebSockets maintain a persistent connection, enabling instant updates.

Real-time features require different patterns than traditional request/response. Pub/sub patterns allow broadcasting messages to multiple clients. Understanding these patterns helps you build engaging, real-time user experiences.

*   **Real-Time:** Features that update instantly without requiring page refresh or polling.
