## ğŸ“– Task Explanations

Loading and error states are essential for good user experience. App Router provides special files (`loading.tsx`, `error.tsx`) that automatically handle these states for routes. Suspense allows you to create more granular loading states, showing parts of the page while other parts are still loading.

Streaming allows Next.js to send HTML to the browser incrementally as it's generated, improving perceived performance. Users see content faster, even if the entire page isn't ready. Understanding these patterns helps you build applications that feel fast and responsive.

## ğŸ“š Dictionary for Developers

*   **Loading State:** A UI state that indicates content is being fetched or processed.
*   **Error Boundary:** A React component that catches errors in its child components and displays a fallback UI.
*   **Suspense:** A React feature that allows you to show fallback UI while components are loading.
*   **Streaming:** Sending HTML to the browser incrementally as it's generated, improving perceived performance.
*   **Perceived Performance:** How fast an application feels to users, not just actual load time.
