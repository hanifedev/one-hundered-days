## ğŸ“– Task Explanations

Server Components are a revolutionary feature that allows React components to render on the server, reducing JavaScript sent to the client and improving performance. They can directly access databases and APIs without exposing sensitive information. Client Components are used when you need interactivity (useState, useEffect, event handlers).

Understanding when to use Server vs Client Components is crucial for building performant Next.js applications. Most of your components should be Server Components, with Client Components used only when necessary for interactivity.

## ğŸ“š Dictionary for Developers

*   **Server Component:** A React component that renders on the server, reducing client-side JavaScript.
*   **Client Component:** A React component that renders on the client and can use React hooks and browser APIs.
*   **'use client' Directive:** A directive that marks a component and its children as Client Components.
*   **Bundle Size:** The amount of JavaScript code sent to the browser, affecting load time and performance.
*   **Server-Side Rendering (SSR):** Rendering React components on the server before sending HTML to the client.
