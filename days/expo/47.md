## üìñ Task Explanations

A Context Provider is typically a component that wraps your app and manages the state you want to share. It uses `useState` or other hooks internally and provides both the state and functions to update it through the context. This pattern keeps your state management logic organized and reusable.

Creating a custom hook for accessing context makes your code cleaner and provides a single place to add error handling (like checking if the context is used outside its provider). Splitting contexts by domain (authentication, theme, shopping cart) keeps your code organized and prevents unnecessary re-renders.

## üìö Dictionary for Developers

*   **Context Provider Component:** A component that wraps part of your app and provides context values to all child components.
*   **Custom Hook:** A reusable function that uses React hooks internally, allowing you to extract and share stateful logic.
*   **Context Splitting:** The practice of creating separate contexts for different domains of your application to improve organization and performance.
*   **Provider Pattern:** A design pattern where a component provides data or functionality to its children through context.
*   **State Management Logic:** The code that handles creating, reading, updating, and deleting state.
