## üìñ Task Explanations

As your app grows, you might notice performance issues with Context API‚Äîevery time context value changes, all consuming components re-render. You can optimize this by splitting contexts (so only relevant components re-render) and using `React.memo()` to prevent unnecessary re-renders of child components.

Context composition allows you to combine multiple contexts in a clean way, which is useful for complex applications. Understanding best practices (like when to use Context vs. local state, how to structure contexts) will help you build maintainable applications that perform well.

## üìö Dictionary for Developers

*   **Context Composition:** The practice of combining multiple contexts together to manage different aspects of application state.
*   **`React.memo()`:** A higher-order component that prevents a component from re-rendering unless its props have changed.
*   **Performance Optimization:** Techniques used to improve the speed and efficiency of an application.
*   **Unnecessary Re-renders:** When a component re-renders even though its props and state haven't changed, wasting resources.
*   **Context Splitting:** The practice of creating separate contexts for different concerns to minimize re-renders.
*   **Mocking:** Creating fake implementations of dependencies (like context) for testing purposes.
