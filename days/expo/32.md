## ğŸ“– Task Explanations

While React Native provides a basic `Button` component, it's quite limited in terms of styling. For most real-world applications, you'll use `TouchableOpacity`, `TouchableHighlight`, or `Pressable` to create custom-styled buttons. These components give you full control over appearance and behavior.

`Pressable` is the most modern and flexible option, providing fine-grained control over press states. `TouchableOpacity` is widely used for its simplicity and visual feedback. Understanding when to use each will help you build better user interfaces.

## ğŸ“š Dictionary for Developers

*   **`Button`:** A basic React Native component for creating buttons, but with limited styling options.
*   **`TouchableOpacity`:** A component that makes its children touchable and provides visual feedback by reducing opacity when pressed.
*   **`TouchableHighlight`:** A component that provides visual feedback by highlighting the pressed area.
*   **`Pressable`:** A modern, flexible component for handling press interactions with fine-grained control over press states.
*   **`onPress`:** A callback function that is called when a touchable component is pressed.
