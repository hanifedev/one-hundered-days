## üìñ Task Explanations

AsyncStorage is React Native's equivalent to web localStorage. It allows you to store data on the device that persists even after the app is closed. This is essential for features like user authentication (storing tokens), user preferences, caching data for offline use, and more.

AsyncStorage is asynchronous (hence the name), so all operations return promises. This is important because storage operations can take time, and you don't want to block your app's UI. Understanding AsyncStorage is fundamental for building apps that work offline and provide a better user experience.

## üìö Dictionary for Developers

*   **AsyncStorage:** A React Native library that provides asynchronous, persistent, key-value storage for your app.
*   **Local Storage:** Storage that exists on the device itself, persisting data even when the app is closed.
*   **Key-Value Storage:** A simple storage model where data is stored and retrieved using unique keys (like a dictionary).
*   **Persistent Storage:** Storage that retains data even after the app is closed or the device is restarted.
*   **Asynchronous Operation:** An operation that doesn't block the execution of other code, typically using promises or callbacks.
