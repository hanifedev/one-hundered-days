## üìñ Task Explanations

Most real-world applications need to fetch data from APIs, which are asynchronous operations. Regular Redux actions are synchronous, so you need middleware (like Redux Thunk) to handle async operations. Thunk allows action creators to return functions that can perform async operations and dispatch actions when they complete.

Redux Toolkit's `createAsyncThunk` simplifies this process by automatically generating the loading, success, and error action types and handling the async logic. This is the modern, recommended way to handle async operations in Redux.

## üìö Dictionary for Developers

*   **Redux Thunk:** Middleware that allows Redux action creators to return functions instead of action objects, enabling async operations.
*   **Middleware:** Code that sits between dispatching an action and the moment it reaches the reducer, allowing you to intercept and modify actions.
*   **Async Action Creator:** An action creator that performs asynchronous operations (like API calls) and dispatches multiple actions.
*   **`createAsyncThunk`:** A Redux Toolkit function that simplifies creating async action creators with automatic loading/error handling.
*   **Thunk Function:** A function that wraps an expression to delay its evaluation, in Redux context: a function returned by an action creator.
