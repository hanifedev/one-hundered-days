## üìö Dictionary for Developers

*   **Code Splitting:** The practice of splitting code into smaller chunks that are loaded on demand.
*   **Lazy Loading:** Loading code or components only when they're needed, reducing initial load time.
*   **Memory Leak:** A bug where memory is allocated but never freed, causing gradual memory consumption.
*   **Bundle Size:** The total size of JavaScript code in your app, affecting download and startup time.
*   **Tree Shaking:** The process of removing unused code from the final bundle.

## üìö Dictionary for Developers

*   **Code Splitting:** Breaking code into smaller chunks that are loaded only when needed.
*   **Debouncing:** Delaying a function call until after a period of inactivity (useful for search inputs).
*   **Throttling:** Limiting how often a function can be called (useful for scroll handlers).
*   **Memory Leak:** When memory is allocated but never freed, causing memory usage to grow over time.
*   **Request Batching:** Combining multiple API requests into a single request to reduce network overhead.
