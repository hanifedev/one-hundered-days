## üìñ Task Explanations

As your app grows, you'll find yourself passing props through many component levels just to get data to a deeply nested component. This is called "prop drilling" and it makes your code harder to maintain. Context API provides a solution by allowing you to share state globally without passing props through every level.

Context API is built into React, so you don't need any external libraries. It's perfect for sharing data that many components need (like user authentication, theme settings, or app-wide configuration). Understanding Context API is the foundation for more advanced state management solutions.

## üìö Dictionary for Developers

*   **Global State:** State that needs to be accessed by multiple components throughout your application.
*   **Prop Drilling:** The practice of passing props through multiple component levels, even when intermediate components don't use them.
*   **Context API:** A React feature that allows you to share data across the component tree without prop drilling.
*   **Context Provider:** A component that makes context values available to all its child components.
*   **`useContext()` Hook:** A React hook that allows components to access context values.
*   **Context Consumer:** A component or hook that reads values from a context.
