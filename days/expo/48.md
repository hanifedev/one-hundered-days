## üìñ Task Explanations

`useReducer` is like `useState`, but for more complex state logic. Instead of directly setting state, you dispatch "actions" that describe what happened, and a "reducer" function determines how the state should change based on that action. This pattern makes state updates more predictable and easier to test.

The reducer pattern is especially useful when you have state with multiple sub-values or when the next state depends on the previous one in complex ways. It's also the foundation for Redux, so learning `useReducer` prepares you for more advanced state management libraries.

## üìö Dictionary for Developers

*   **`useReducer` Hook:** A React hook that manages state using a reducer function, useful for complex state logic.
*   **Reducer Function:** A pure function that takes the current state and an action, and returns the new state.
*   **Action:** An object that describes what happened and contains information needed to update the state (typically has a `type` and optional `payload`).
*   **Dispatch:** A function returned by `useReducer` that sends actions to the reducer to trigger state updates.
*   **Action Type:** A string constant that identifies the type of action being performed (e.g., `'ADD_TODO'`, `'DELETE_TODO'`).
*   **Pure Function:** A function that always returns the same output for the same input and has no side effects.
